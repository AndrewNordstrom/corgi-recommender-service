Configure Elk with Corgi Proxy

Quick Start

Step 1: Enter Server Address

When prompted for the server in Elk, enter:

localhost:5003

Do not include http:// or https://

Step 2: Authenticate

If prompted for login credentials, use anything as email/password:

  • Email: anything@example.com
  • Password: anything

The proxy will automatically authenticate you with the demo token.

Step 3: Configure Manually if Steps 1-2 Don't Work

If direct login doesn't work, you'll need to:

  1. Create a local access token manually: _Tb8IUyXBZ5Y8NmUcwY0-skXWQgP7xTVMZCFkqvZRIc
  2. Configure your instance URL manually: https://localhost:5003

Testing the API Directly

To verify the proxy is working, test the API directly:

curl -k -H "Authorization: Bearer _Tb8IUyXBZ5Y8NmUcwY0-skXWQgP7xTVMZCFkqvZRIc" https://localhost:5003/api/v1/accounts/verify_credentials

You should see a JSON response with account information.

Common Issues

  • Instance not found: Make sure the proxy server is running on port 5003
  • Authentication failed: Ensure you're using the correct token
  • Elk not connecting: Try entering just localhost as the server address
